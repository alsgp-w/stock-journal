# 주식용 트레이딩 저널 기획안

## 1. 목표

기존 코인용 트레이딩 저널을 주식 투자 기록용으로 재구성한다.

핵심 목표:

- 매매 1건 단위 기록
- 하루 단위 수익 집계
- 달력에서 일별 수익 확인
- 월별/종목별/전략별 성과 분석
- 복기하기 쉬운 저널 구조 만들기

---

## 2. 참고 사이트 기준 현재 구조

참고 사이트는 아래 4개 블록 중심이다.

- 진입 수량 및 레버리지 계산기
- 금일 매매 기록 입력 폼
- 누적 매매 기록 목록
- 승률 통계 차트

이 구조는 코인 선물 거래에는 맞지만, 주식 투자에는 아래 항목들이 더 중요하다.

- 매수/매도 구분
- 종목명 + 티커
- 보유 수량
- 평균단가 / 매도가
- 수수료 / 세금
- 당일 실현손익
- 보유 포지션 여부
- 매매 사유와 전략 태그

---

## 3. 주식용으로 바뀌어야 할 핵심 포인트

### 3-1. 레버리지 중심 계산기는 제거 또는 교체

주식용에서는 `레버리지 계산기`보다 아래 기능이 더 유용하다.

- 매수 가능 수량 계산기
- 손절 기준 리스크 계산기
- 목표가 도달 시 예상 수익 계산기

추천:

- 1순위: `리스크 기반 매수 수량 계산기`
- 2순위: `목표가/손절가 기반 손익 계산기`

예시 입력:

- 투자금
- 매수가
- 손절가
- 목표가
- 허용 손실률(%)

예시 출력:

- 권장 매수 수량
- 손절 시 예상 손실
- 목표가 도달 시 예상 수익
- 손익비(R/R)

### 3-2. 포지션 개념 변경

코인 사이트의 `LONG/SHORT`는 주식에 그대로 맞지 않는다.

주식용에서는 아래 중 하나로 바꾸는 것이 좋다.

- `매수 / 매도`
- 또는 `신규매수 / 추가매수 / 부분매도 / 전량매도`

추천:

- 기본형: `매수`, `매도`
- 확장형: `신규매수`, `추가매수`, `부분매도`, `전량매도`

### 3-3. 수익금만 기록하지 말고 거래 구조를 함께 저장

단순히 수익금만 적으면 나중에 복기가 약해진다.

주식용 로그에는 아래 데이터가 있으면 좋다.

- 거래일
- 종목명
- 티커
- 시장 구분(KOSPI/KOSDAQ/NYSE/NASDAQ 등)
- 거래 유형(매수/매도)
- 매수 단가
- 매도 단가
- 수량
- 총 매수금액
- 총 매도금액
- 수수료
- 세금
- 실현손익
- 수익률(%)
- 전략 태그
- 매매 근거
- 심리 상태
- 복기 메모
- 차트 이미지 또는 링크

## 4. 추천 정보 구조

### 4-1. 메인 화면 구성

메인 화면은 아래 5개 섹션으로 구성하는 것을 추천한다.

1. 상단 요약 대시보드
2. 주식용 매매 입력 폼
3. 수익 달력
4. 누적 거래 기록
5. 통계/분석 영역

### 4-2. 상단 요약 대시보드

한눈에 보이는 카드형 지표:

- 오늘 실현손익
- 이번 주 손익
- 이번 달 손익
- 누적 손익
- 승률
- 평균 수익 / 평균 손실
- 최대 수익 거래
- 최대 손실 거래

추천 이유:

사용자가 사이트에 들어오자마자 "오늘 얼마 벌었는지", "이번 달 흐름이 어떤지"를 바로 볼 수 있다.

### 4-3. 주식용 매매 입력 폼

필수 입력:

- 날짜
- 종목명
- 티커
- 거래 유형
- 수량
- 매수 단가
- 매도 단가
- 수수료/세금
- 실현손익

선택 입력:

- 전략 태그
- 진입 근거
- 실수한 점
- 심리 상태
- 스크린샷 링크
- 메모

입력 UX 추천:

- 종목명과 티커는 분리
- 손익은 자동 계산되게 만들기
- 직접 수정도 가능하게 허용
- 전략 태그는 버튼형 선택 UI

## 5. 가장 중요한 기능: 수익 달력

### 5-1. 달력 기능 목표

사용자는 달력에서 날짜별 손익을 바로 보고 싶어 한다.

따라서 달력은 단순 일정 캘린더가 아니라 `일별 손익 히트맵` 역할을 해야 한다.

### 5-2. 달력에 표시할 내용

각 날짜 칸에 아래 내용을 표시한다.

- 해당 날짜 총 손익 금액
- 거래 횟수
- 색상 상태

색상 규칙 예시:

- 수익일: 초록
- 손실일: 빨강
- 보합일: 회색
- 거래 없음: 어두운 기본색

예시:

- `+125,000원`
- `-34,500원`
- `0원`

### 5-3. 날짜 클릭 시 상세 보기

달력의 날짜를 클릭하면 그날 거래 목록을 오른쪽 패널 또는 모달로 보여준다.

상세 내용:

- 총 손익
- 총 거래 횟수
- 거래한 종목 목록
- 각 거래별 수익/손실
- 매매 메모

### 5-4. 달력 필터

달력 위에 필터를 두면 훨씬 실용적이다.

- 월 선택
- 연도 선택
- 특정 종목만 보기
- 전략 태그 기준 보기

### 5-5. 달력 아래 보조 요약

달력과 함께 아래 수치도 보여주면 좋다.

- 이번 달 수익일 수
- 이번 달 손실일 수
- 가장 많이 번 날
- 가장 많이 잃은 날
- 하루 평균 손익

---

## 6. 통계 기능은 이렇게 바꾸는 것이 좋다

기존 사이트는 승/무/패 중심인데, 주식용은 손익 구조 분석이 더 중요하다.

추천 통계:

- 월별 손익 막대 차트
- 누적 손익 라인 차트
- 종목별 손익 순위
- 전략별 승률
- 요일별 성과
- 시간대별 성과
- 평균 보유 기간

우선순위 추천:

- 1차: 누적 손익, 월별 손익, 승률
- 2차: 종목별 성과, 전략별 성과
- 3차: 요일별/시간대별 분석

## 7. 데이터 구조 제안

거래 1건 기준 예시:

```json
{
  "id": 1710000000000,
  "date": "2026-04-22",
  "stockName": "삼성전자",
  "ticker": "005930",
  "market": "KOSPI",
  "tradeType": "BUY",
  "quantity": 10,
  "buyPrice": 71200,
  "sellPrice": 72800,
  "fees": 1500,
  "tax": 300,
  "profit": 14200,
  "profitRate": 1.99,
  "strategy": "돌파매매",
  "reason": "전고점 돌파 후 거래량 증가",
  "psychology": "조급하지 않고 계획대로 진입",
  "memo": "목표가 도달 후 절반 익절",
  "imageUrl": ""
}
```

하루 단위 집계 데이터는 별도 저장 없이 거래 데이터로 계산하는 방식이 단순하다.

예:

- 같은 날짜의 `profit` 합계
- 해당 날짜 거래 수 count
- 양수/음수 여부에 따라 달력 색상 결정

## 8. MVP 기능 우선순위

처음부터 너무 크게 만들기보다 아래 순서로 가는 것이 좋다.

### 1단계 MVP

- 거래 입력/수정/삭제
- 로컬 저장
- 거래 목록
- 오늘/이번 달 손익 요약
- 일별 수익 달력

### 2단계

- 종목별 필터
- 전략 태그
- 월별 통계 차트
- 날짜 클릭 상세 보기

### 3단계

- CSV 내보내기
- 데이터 백업/복원
- 다크/라이트 테마
- 모바일 최적화 강화

## 9. 화면 디자인 방향

참고 사이트는 다크 테마 기반 단일 페이지 구조다.

주식용으로 바꿀 때도 단일 페이지는 유지하되, 정보 우선순위를 아래처럼 바꾸는 게 좋다.

- 상단: 오늘 손익, 이번 달 손익, 승률
- 중단 좌측: 거래 입력
- 중단 우측: 수익 달력
- 하단: 거래 목록과 통계

디자인 포인트:

- 수익은 선명한 초록, 손실은 선명한 빨강
- 달력 칸 hover 효과
- 모바일에서는 입력 폼 > 요약 > 달력 > 목록 순서
- 숫자는 천 단위 구분
- 원화/달러 표시 선택 가능하면 좋음

## 10. 추천 기능명 변경안

기존 코인용 용어를 주식용으로 바꾸면 더 자연스럽다.

- `Trading Journal` -> `Stock Trading Journal` 또는 `주식 매매일지`
- `LONG/SHORT` -> `매수/매도`
- `수익금($)` -> `실현손익(원)`
- `매매 근거` -> `진입 근거`
- `비고` -> `복기 메모`
- `매매 통계` -> `성과 분석`

---

## 11. 구현 방식 제안

가볍게 시작하려면 아래 방식이 적합하다.

- HTML
- CSS
- JavaScript
- `localStorage` 저장

이유:

- 참고 사이트 구조와 가장 잘 맞음
- GitHub Pages 배포 쉬움
- 개인 기록용 사이트로 충분히 빠름

나중에 확장하고 싶다면:

- React 또는 Next.js 전환
- Firebase/Supabase 저장
- 로그인 기능 추가

---

## 12. 최종 추천 방향

가장 추천하는 형태는 아래와 같다.

`주식 매매 입력 + 일별 수익 달력 + 월별 성과 분석` 중심의 개인용 저널

핵심 차별점:

- 단순 일지보다 "하루에 얼마 벌었는지"를 바로 확인 가능
- 거래 1건 기록과 하루 총합을 동시에 볼 수 있음
- 달력 중심이라 복기가 빠름
- 주식 투자 습관 분석에 유리함
